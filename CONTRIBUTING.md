The following is a set of guidelines for contributing to this project. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## Code of Conduct

This project and everyone participating in it is governed by the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How Can I Contribute?

### Reporting Bugs

- **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/zule-ai/recaptcha-classifier/issues).
- If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/zule-ai/recaptcha-classifier/issues/new).

### Pull Requests

1. **Fork the repo** and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code lints (`npm run lint`).
5. format your code (`npm run format`).

## Styleguides

### JavaScript Styleguide

- We use [ESLint](https://eslint.org/) with [Standard JS](https://standardjs.com/).
- We use [Prettier](https://prettier.io/) for formatting.

### Git Commit Messages

- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.
